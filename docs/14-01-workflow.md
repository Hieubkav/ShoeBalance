> **Tưởng tượng bạn là chủ tiệm giày:**
>
> Bạn muốn trả lời câu hỏi đơn giản: "Cần đặt thêm bao nhiêu đôi giày từ Trung Quốc?"

> Để trả lời, bạn cần biết:
> - Còn bao nhiêu đôi trong kho?

## Bảng Đối Chiếu Thuật Ngữ

| Thuật ngữ | Giải thích dễ hiểu |
|-----------|-------------------|
| **SKU** | Mã riêng của từng đôi giày (VD: ABC123-42 = mẫu ABC123, size 42) |
| **Tồn kho hiện tại** | Số đôi đang có trong kho ngay bây giờ |
| **Hàng đang về** | Số đôi đã đặt mua, đang ship về |
| **Tồn kho tối thiểu** | Số đôi "không được để dưới mức này" để tránh hết hàng |
| **Xuất kho** | Số đôi đã bán ra trong 30 ngày vừa qua |
| **Tỉ suất bán** | Trung bình mỗi ngày bán được bao nhiêu đôi = Xuất kho ÷ 30 |
| **Ngưỡng nhập** | Số tối thiểu để đáng nhập (VD: > 12 đôi mới nhập) |
| **Size nữ** | Size 36-39 |
| **Size nam** | Size 40-45 |

```
Tỉ suất bán = Tổng xuất kho (CẢ MÃ sản phẩm, không phải từng size) ÷ 30 ngày
```

> Ví dụ: Mã ABC bán được 45 đôi trong 30 ngày
> → Tỉ suất = 45 ÷ 30 = 1.5 (bán 1.5 đôi/ngày)

#### B1. Bán Chậm (Tỉ suất < 0.4)

> **Như kệ sách ít người đọc:** Không cần để nhiều

**Điều kiện áp dụng:** 
- sellRate < 0.4 (bán dưới 12 đôi/tháng)
- VÀ Tồn hiện tại < 13 đôi

| Size | Tồn Tối Thiểu | Lý do |
|------|---------------|-------|
| 40   | 3 đôi         | Size biên, ít người mua |
| 41   | 5 đôi         | Size chính |
| 42   | 5 đôi         | Size chính - phổ biến nhất |
| 43   | 5 đôi         | Size chính |
| 44   | 3 đôi         | Size biên |
| 45   | 2 đôi         | Size biên, rất ít người mua |

**Tổng: 23 đôi cho 1 mẫu bán chậm**

---

#### B2. Bán Nhanh (Tỉ suất >= 0.4)

> **Như kệ best-seller:** Càng bán nhanh, càng để nhiều

**Điều kiện áp dụng:** 
- sellRate >= 0.4 (bán từ 12 đôi/tháng trở lên)
- VÀ Tồn hiện tại < (15 + 12 × sellRate)

**Công thức tính tổng lý tưởng:**
```
Tổng tồn lý tưởng = 24 + 12 × sellRate
```

> Ví dụ: Tỉ suất = 1.5
> → Tổng = 24 + 12 × 1.5 = 42 đôi

**Phân bổ theo size (ưu tiên size chính):**

```
┌────────────────────────────────────────────────────────────────┐
│              PHÂN BỔ SIZE CHO MẪU BÁN NHANH                    │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  Bước 1: Tính baseStock = Tổng × 20.58%                       │
│  Bước 2: Tính edgeStock = baseStock - 2                       │
│                                                                │
│  Bước 3: Áp dụng giới hạn size biên                           │
│          Nếu edgeStock > 4 → edgeStock = 4                    │
│          Phần thừa (excess) chuyển cho size chính             │
│                                                                │
│  Bước 4: Phân bổ theo ưu tiên                                 │
│  ┌─────────────────────────────────────────────────────┐      │
│  │  SIZE CHÍNH (nhận thêm phần thừa từ size biên)     │      │
│  │  • Size 42: (baseStock + excess) × 1.2             │      │
│  │  • Size 41: (baseStock + excess) × 1.0             │      │
│  │  • Size 43: (baseStock + excess) × 0.8             │      │
```

> **Tại sao giới hạn size biên?**
> Size 40, 44, 45 ít người mua hơn. Nếu để nhiều sẽ tồn kho lâu.
> Phần thừa sẽ chuyển sang size chính (42 > 41 > 43).

## Ví Dụ Thực Tế

### Ví dụ 1: Size Nữ - Mẫu ABC123-37

```
Dữ liệu:
- Tồn hiện tại: 2 đôi
- Đang về: 0 đôi  
- Tháng rồi bán (size 37): 6 đôi

Tính toán:
- Tồn tối thiểu = MIN(6, 8) = 6 đôi
- Cần nhập = 6 - 2 - 0 = 4 đôi ✅
```

### Ví dụ 2: Size Nam Bán Chậm - Mẫu XYZ-42

```
Dữ liệu:
- Tồn hiện tại: 2 đôi
- Đang về: 0 đôi
- Tổng xuất kho (cả mã XYZ): 9 đôi
- Tỉ suất = 9 ÷ 30 = 0.3 (< 0.4 → Bán chậm)

Tính toán:
- Size 42 bán chậm → Tồn tối thiểu = 5 đôi
- Cần nhập = 5 - 2 - 0 = 3 đôi ✅
```

### Ví dụ 3: Size Nam Bán Nhanh - Mẫu HOT-41

```
Dữ liệu:
- Tồn hiện tại: 3 đôi
- Đang về: 2 đôi
- Tổng xuất kho (cả mã HOT): 45 đôi
- Tỉ suất = 45 ÷ 30 = 1.5 (>= 0.4 → Bán nhanh)

Kiểm tra điều kiện:
- 15 + 12 × 1.5 = 33 đôi
- Tồn hiện tại (3) < 33 → ĐỦ ĐIỀU KIỆN

Tính toán:
- Tổng lý tưởng = 24 + 12 × 1.5 = 42 đôi
- Size 41 (size chính) = 42 × 20.58% × 1.0 ≈ 9 đôi
- Cần nhập = 9 - 3 - 2 = 4 đôi ✅
```
